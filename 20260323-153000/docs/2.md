# 2. Functional Design - Maximo PM (Safety Inspection Based) Work Order Automation

## 2.1. Business Process

1. Operations and safety engineering team defines Safety Inspection templates and associated SM Planned Maintenance (PM).
2. PM issauance generates safety inspection work orders at defined intervals (schedule).
3. Safety inspection is conducted in MAS 8.x by technicians or inspectors.
4. Inspector completes the inspection checklist with pre-defined response values (PASS, FAIL, NOT APPLICABLE) in Maximo.
5. On completion, the system automatically triggers an Automation Script on the Work Order (WORKORDER) object.
6. The script evaluates the active inspection results associated with the PM and, for each inspection result, if ANY value = FAIL, it creates a work order.
7. One work order is created per inspection result to achieve fine-grain gramularity and downstream safety management attribution.
8. The work order isassigned to the appropriate asset, location, and environment.
9: Priority of the work order is automatically determined based on inspection item criticality.

## 2.2 Features & Requirements

- Feature: Deterministic mapping between Safety Inspection results and work orders in Maximo. Requirement: Inspection results are persistently linked to the work order notes and history.
- Feature: One work order per safety inspection result. Requirement: Configurable justification for one-versus-multiple work orders py inspection result.
- Feature: Criticality-based prority mapping (Major/minor/critical). Requirement: Standardized mapping table to translate inspection criticality values to Maximo work order priority (1-3).
- Feature: Audit trace and compliance reporting. Requirement: Visibility into individual inspection results, work orders, and their status in Maximo.

## 2.3 Scope

In scope are:

- Maximo Application Suite 8.x encluding Work Order, Asset management, SMPM, Safety Inspections, and Preventive Maintenance.
- Preventive Maintenance PM templates with associated Safety Inspection templates.
- Only safety type inspections are covered. Other inspection types (eg. Asset condition) may be added later but are out of scope for this initial release.
- Under one SAP/Tenant.

## 2.4 User Stories & Use Cases

- Safety Manager Persona. As a safety manager, I want to ensure that any failure during our preventive safety inspections quickly generates a engineering work order so that we can risk_base iterate faster.
- Operations Manager. I want visibility on major and critical safety failures, and the ability to look at trends per location, asset, and technician.
- HE Safety Inspector. I need a simple, exact work flow from mechanically critical findings in the check list to an actionable work order jin.

## 2.5 Assumptions

- Safety Inspection response list is configured with values OK (PASS), FAIL, NOT APPLICABLE in the Inspection checklist.
- Each inspection result is linked to exactly one work order only, avoiding duplicate tasks.
- Priority mapping is based on Inspection Item criticality field with mapping: Critical = 1, Major = 2, Minor = 3.
- ORG already has PM and Inspection template configured, no schema changes needed.
